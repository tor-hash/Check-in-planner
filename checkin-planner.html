Live Google Calendar (Apps Script) Ikke konfigureret

Hver manager deployer sit eget Apps Script og indsætter sin egen URL nedenfor. Du behøver kun udfylde din egen — andre managers indsætter deres URL i deres egen browser.
Medarbejdere skal ikke deploye noget — de skal blot dele deres Google Calendar med dig (Calendar settings → Share with specific people → "See only free/busy").

Sådan opretter du Apps Script (klik for guide)
  1. Gå til script.google.com og opret et nyt projekt.
  2. Slet det tomme function myFunction() og indsæt koden nedenfor.
  3. Aktivér Calendar API: i venstre sidebar, klik Services (+ ikon). Find Google Calendar API, lad identifier være Calendar, klik Add. Det er nødvendigt for FreeBusy API'et — uden det får du 0 OK · 21 fejl.
  4. Klik Deploy → New deployment → Web app. Sæt Execute as: Me og Who has access: Anyone. Klik Deploy. Autorisér scope første gang (Calendar read).
  5. Kopiér Web app URL (slutter på /exec) og indsæt i feltet her på siden. Klik Test forbindelse, så Refresh.
  6. Hvis du har en eksisterende deployment: Deploy → Manage deployments → ✎ → New version → Deploy. URL'en kan forblive den samme.
  7. Domæne-deling: i Google Workspace er free/busy som standard delt internt på domænet. Hvis nogle medarbejdere stadig fejler, skal de manuelt dele deres kalender (Calendar Settings → Share with specific people → See all event details til din email).
// Kræver: Services → Google Calendar API tilføjet (identifier: Calendar)
function doGet(e) {
  const params = e.parameter;
  const callback = params.callback;
  const calId = params.calendar || "primary";
  const start = new Date(params.start || new Date());
  const end = new Date(params.end || new Date(Date.now() + 14*864e5));
  const realId = (calId === "primary") ? Session.getActiveUser().getEmail() : calId;

  let payload;
  try {
    // FreeBusy API - virker for alle delte kalendere på domænet
    const fb = Calendar.Freebusy.query({
      timeMin: start.toISOString(),
      timeMax: end.toISOString(),
      items: [{ id: realId }]
    });
    const cal = fb.calendars[realId] || Object.values(fb.calendars)[0];
    if (cal && cal.errors && cal.errors.length) {
      payload = { error: cal.errors[0].reason + " (" + realId + ")" };
    } else {
      const busy = (cal && cal.busy ? cal.busy : []).map(b => ({
        start: b.start, end: b.end, summary: ""
      }));
      payload = { calendar: realId, count: busy.length, busy: busy };
    }
  } catch (err) {
    payload = { error: "Script fejl: " + String(err) };
  }

  if (callback) {
    return ContentService.createTextOutput(callback + "(" + JSON.stringify(payload) + ")")
      .setMimeType(ContentService.MimeType.JAVASCRIPT);
  }
  return ContentService.createTextOutput(JSON.stringify(payload))
    .setMimeType(ContentService.MimeType.JSON);
}

Sikkerhed: URL'en giver enhver med linket adgang til at læse de kalendere du har givet scriptet adgang til. Hold den privat.
